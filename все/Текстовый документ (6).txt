1) Правило №1: работаем патчами по 1 файлу

Ты кидаешь это в ChatGPT каждый раз:

PROMPT (копируй как есть):

Контекст: я делаю Telegram-бота. Деплой на Render. Python.

Работай ТОЛЬКО итерациями по 1 файлу.

Сначала: 5 буллетов “что сломано/что делаем”.

Потом: дай diff-патч для ОДНОГО файла (без воды).

Потом: команды проверки (1–3 команды).

Стоп. Жди моего “далее”.

Формат ответа строго:

План (5 пунктов)

Патч: diff ...

Команды: bash ...

2) Как давать входные данные (чтобы он не “думал вечность”)

Ты всегда отправляешь только:

1 лог/ошибку (или 1 симптом)

1–2 файла целиком (или кусок файла где ошибка)

что ты ожидаешь (“кнопка не работает”, “callback падает”, “Render крашится”)

Пример запроса:
“Вот лог Render + файл integration.py. Исправь так, чтобы не падало, и добавь verify-скрипт.”

3) Сейв прогресса (на случай обрыва)

Перед тем как вставлять большой текст/файл:

Ctrl+A → Ctrl+C в буфер

или вставляй в notes.txt рядом с проектом

4) “Режим коротких ответов” (чтобы не рвалось соединение)

Если видишь, что он начинает писать простыню — сразу пиши:
“СТОП. Только патч одного файла и команды. Без объяснений.”

5) Мини-скрипт, который спасает жизнь (локально)

Сделай файл scripts/verify_project.py и гоняй всегда перед пушем/деплоем:

python -m compileall .

python -m pip check

python -m pytest -q (если есть тесты)
